# FoodMate 文档整理与代码审计实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将项目级文档统一迁入中文 `docxs/` 目录，补齐双运行时实施、测试、运维和代码审计文档，并保持所有引用有效。

**Architecture:** 根目录只保留精简 `README.md`，`docxs/文档索引.md` 作为文档导航与权威来源表。现有长文档按产品、设计、架构、契约、数据、实现、项目、参考和归档分类迁移；新增文档只补现有体系无法回答的实施问题，不修改生产代码。

**Tech Stack:** Markdown、Git、PowerShell、Maven Wrapper、npm/Vite/ESLint/Vitest。

## Global Constraints

- 根目录只保留项目级 `README.md`，其余项目级 Markdown 进入 `docxs/`。
- `docxs/` 下目录和文档名称使用中文；ADR 编号保留英文前缀。
- `.workbuddy/`、`frontend/node_modules/`、构建产物、源码和构建配置不移动、不修改。
- 不回改 Flyway V1，不创建 Python 工程，不修改前端或后端生产代码。
- 当前实现、目标设计和历史参考必须显式区分。
- 所有注册工具、SQL Guard 和实际执行统一归 Java；Python 负责 Agent 推理与编排。
- 文件移动前必须确认目标不存在，禁止覆盖有效文件。
- 每项验证必须报告真实结果；前端 lint 和 test 当前预期失败。
