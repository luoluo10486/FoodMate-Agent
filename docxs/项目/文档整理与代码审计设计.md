## 1. 目标

本次工作解决三个问题：

1. 将迁移前分散在根目录和旧文档目录的项目文档统一整理到 `docxs/`。

## 5. 新增文档职责

### 5.1 文档索引

`docxs/文档索引.md` 是文档体系的唯一导航页，记录：

### 5.2 当前实现审计与完善计划

`docxs/实现/当前实现审计与完善计划.md` 必须基于实际文件和命令输出，包含：

- 已真实实现能力。
- mock、占位和空骨架。
- 前端、Java 后端和 Python Runtime 缺口。
- P0、P1、P2 优先级。
- 涉及模块、依赖关系和验收方式。
- 已执行命令、通过项、失败项和未验证项。

不得把构建通过等同于功能完成，也不得把目标架构描述成当前实现。

## 6. 代码审计基线

### 6.1 前端

当前结论：前端是可运行的 Phase 1-2 静态和 mock 原型，不是已接通后端的业务前端。

- `npm run build` 通过。
- `npm run lint` 因 35 个 warning 和零 warning 门槛失败。
- `npm run test` 因没有测试文件失败。
- 所有 service 仍指向 mock，无真实 HTTP/SSE 客户端。
- 没有真实认证状态、token 恢复、refresh、路由守卫或 RBAC。
- 会话回放 Hook 未按 `session_id` 重置初始化标记，切换路由时存在沿用上一会话状态的风险。
- 部分可见按钮没有行为或禁用反馈。

前端 P0：修复 lint、建立最小测试、修复会话切换状态、处理无行为控件。真实后端联调前再实现 API Client、认证状态机和 AgentRun/SSE Client。

### 6.2 Java 后端

当前结论：Java 后端是可编译、可启动的基础骨架，不是可用业务控制面。

- Maven Reactor 构建成功，现有 29 个测试通过。
- shared、api、infra 基础能力和 Flyway V1 已建立。
- domain、application、gateway-client、tool、worker 及旧 Agent 模块主要仍是标记类。
- 没有 PO/Mapper/Repository、认证/RBAC、会话、消息、AgentRun、Runtime Client、真实 SSE、工具执行或 SQL Guard。
- application 仍依赖冻结的旧 Agent Maven 模块。
- V1 缺少跨运行时事件去重、dispatch/attempt、调用幂等和正确的模型调用唯一键。
- 软删除恢复逻辑只有代理测试，无法证明真实 SQL 能更新已删除行。
- Flyway 只有文本结构测试，尚未在 PostgreSQL/Testcontainers 中执行。

Java P0：验证并修复软删除恢复、增加真实 PostgreSQL 迁移测试、建立 V2 设计、收缩旧模块依赖并实现 Runtime Client 契约基线。

### 6.3 Python Runtime

当前仓库没有 `agent-runtime/`、`pyproject.toml`、项目虚拟环境或 Python 测试，所有 Python 内容均属于目标设计。

## 7. 内容收敛规则

- 每个主题只保留一个权威详细文档。
- 摘要文档只给结论和链接，不复制完整章节。
- 当前状态、目标设计和历史参考必须显式标记。
- 删除 `code_review.md`、`resume_optimizer.md` 等与 FoodMate 无关的示例。
- 历史模型网关 SQL 和旧 Java Agent 方案移入参考或归档，并标明不实施。
- 不删除仍有效的产品、接口、数据库和验收信息。

- 不回改已发布的 Flyway V1；V2 只写设计，不创建迁移代码。
- 任何无法验证的代码能力标记为“未验证”，不推断完成。
