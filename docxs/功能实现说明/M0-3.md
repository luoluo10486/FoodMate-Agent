## 2. 接口清单

本阶段不新增业务接口，影响所有 `/api` 写请求及应用启动配置。健康检查接口继续由 Spring Actuator 提供。

## 3. 实现逻辑

应用启动时按 profile 读取配置。local、dev、prod 的 Flyway 均关闭。Cookie 的 Secure、SameSite、Path 等属性由配置控制；认证写请求必须通过同源和 CSRF 校验。Runtime Service JWT 的开关、密钥、kid 和契约版本由环境变量注入，缺失关键配置时由启动校验器拒绝启动或报告不可用。

## 4. 外部依赖与配置

敏感值仅从 `.env` 或部署 Secret 注入，包括数据库、QQ SMTP、MinIO、Runtime JWT 和模型供应商密钥。`.env` 被 Git 忽略，`.env.example` 不含真实秘密。

## 5. 安全与隐私

- 禁止跨源 API 写操作。
- 认证 Cookie 不暴露会话令牌给 JavaScript。
- 不在日志、响应或文档中输出密码、令牌哈希、SMTP 授权码、MinIO 密钥或模型密钥。
- Runtime 不可用时，触发 Agent 执行的接口必须返回稳定的 `RUNTIME_UNAVAILABLE`，不能写入虚假的派发成功状态。
