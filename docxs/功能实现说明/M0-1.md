## 2. 功能概述

通过 Docker Compose 提供本地 PostgreSQL、Redis 和 MinIO，并通过人工 SQL 建立 FoodMate 核心表及 M1-1 所需账户隐私表。Flyway 默认关闭，数据库变更由人工脚本控制和校验。

## 3. 接口清单

本阶段没有新增业务 HTTP 接口。基础设施通过数据库连接、Redis 连接和 MinIO S3 兼容接口提供能力。

## 4. 实现逻辑

1. Docker Compose 启动 PostgreSQL、Redis 和 MinIO。
2. 先执行数据库备份，再执行 V1 核心建表 SQL 和 V2 账户隐私 SQL。
3. 执行校验 SQL，确认表、索引、约束和 `superadmin` 角色存在。
4. 使用应用配置连接 PostgreSQL、Redis 和 MinIO；Flyway 不自动执行迁移。

## 5. 涉及的数据库表

V1 建立核心用户、资料、头像、会话、消息、运行时、审计等表；V2 增加或调整：

- `users`
- `user_profiles`
- `user_avatar_assets`
- `user_auth_sessions`
- `password_reset_tokens`
- `data_export_jobs`
- `account_deletion_jobs`

## 6. 外部依赖与配置

| 依赖 | 用途 | 不可用时行为 |
|---|---|---|
| PostgreSQL | 业务持久化 | local 真实模式无法启动或提供持久化能力 |
| Redis | 缓存/基础设施预留 | 由运行配置决定是否阻断启动 |
| MinIO | 私有头像和导出文件对象存储 | 头像、导出任务不可执行 |

敏感配置只从 `.env` 读取，`.env` 不提交 Git；`.env.example` 仅作为变量模板。
