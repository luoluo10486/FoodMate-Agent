## 3. 实现逻辑

### 3.1 按环境读取配置

- Spring Boot 启动时按 `local`、`dev`、`prod` profile 合并配置。
- 数据库配置从环境变量或部署 Secret 注入。
- 邮件配置从环境变量或部署 Secret 注入。
- MinIO 配置从环境变量或部署 Secret 注入。
- Runtime JWT 配置从环境变量或部署 Secret 注入。
- 各环境均关闭 Flyway。
- 应用启动不会未经确认自动执行数据库变更。

### 3.2 Cookie、同源与 CSRF 安全策略

- `AuthController` 根据 `foodmate.security.cookie-secure` 设置 Cookie 的 Secure 属性。
- Cookie 固定使用 `SameSite=Lax`。
- Cookie 固定使用 `Path=/`。
- 会话 Cookie 设置 HttpOnly。
- CSRF Cookie 保持可读，供前端写入请求头。
- `CsrfProtectionFilter` 放行安全方法。
- Filter 放行明确列出的公开认证接口。
- 其他写请求必须通过会话、Origin 和 CSRF 校验。

### 3.3 Runtime Service JWT 配置与启动门禁

- Runtime 调用由 `RuntimeGatewayController` 接收。
- `ServiceJwt.verify` 解码 JWT。
- Service 校验 EdDSA 签名。
- Service 校验 `iss`、`aud`、必需 scope、`exp` 和 `jti`。
- 任一 JWT 校验失败都拒绝请求。
- JWT 开关开启但缺少公私钥时，配置校验器阻止启动。
- 缺少 kid 或契约版本时，配置校验器阻止启动。
- Runtime 不可用时返回 `RUNTIME_UNAVAILABLE`。
- Runtime 不可用时不能写入虚假的成功状态。

## 4. 外部依赖与配置

敏感值仅从 `.env` 或部署 Secret 注入，包括数据库、QQ SMTP、MinIO、Runtime JWT 和模型供应商密钥。`.env` 被 Git 忽略，`.env.example` 不含真实秘密。

## 5. 安全与隐私

- 禁止跨源 API 写操作。
- 认证 Cookie 不暴露会话令牌给 JavaScript。
- 不在日志、响应或文档中输出密码、令牌哈希、SMTP 授权码、MinIO 密钥或模型密钥。
- Runtime 不可用时，触发 Agent 执行的接口必须返回稳定的 `RUNTIME_UNAVAILABLE`，不能写入虚假的派发成功状态。
