# M1-1 账户、授权与个人数据

## 1. 实现状态

核心账户与个人数据服务已接入 PostgreSQL、QQ SMTP 和 MinIO。密码、会话、头像、导出、注销和 RBAC 后端接口可用；前端个人资料页已接入真实接口。管理后台目前只有用户查询具备真实后端接口，其余管理模块在 real 模式明确显示未接入，不再展示 mock 或伪造成功。

## 3. 实现逻辑

### 3.1 密码重置

- `AuthController.requestReset` 接收邮箱。
- Controller 调用 `UserAccountService.createPasswordResetToken`。
- Service 先让该用户之前未使用的令牌失效。
- Service 生成新的随机令牌。
- 只有 SHA-256 哈希写入 `password_reset_tokens`。
- 令牌记录保存用户 ID。
- 令牌有效期为 900 秒。
- 接口返回统一成功结果，不暴露邮箱是否存在。
- 配置 SMTP 后，Controller 使用 `FOODMATE_WEB_BASE_URL` 拼接重置地址。
- 邮件同时生成 UTF-8 HTML 正文和纯文本正文。
- 确认接口调用 `resetPassword`。
- Service 只接受未使用且未过期的令牌。
- 密码更新成功后标记令牌已消费。
- 密码更新成功后撤销用户全部会话。

### 3.2 会话与密码安全

- 修改密码接口先验证当前密码。
- Service 按 PBKDF2 格式生成新密码哈希。
- Service 更新 `users.password_hash`。
- Service 批量更新 `user_auth_sessions.revoked_at`。
- 修改密码后旧设备全部退出。
- 单个会话撤销带用户 ID 条件。
- 用户不能撤销其他用户的会话。
- 管理员撤销全部会话时写入操作者信息。

### 3.3 头像存储

- Controller 限制头像类型为 PNG/JPEG/WebP。
- Controller 限制头像大小不超过 2 MB。
- `PersonalDataService.uploadAvatar` 清洗用户文件名。
- Service 按 `avatars/{userId}/{id}-文件名` 生成对象 key。
- Service 将文件上传到 MinIO。
- 上传成功后将旧 active 头像标记为 replaced。
- Service 在 `user_avatar_assets` 写入新对象记录。
- 删除时先查询 active 对象 key。
- Service 尝试删除 MinIO 对象。
- Service 再将数据库记录标记为 deleted。

### 3.4 个人数据导出

- 创建导出接口只插入 `data_export_jobs(status='queued')`。
- 请求线程不直接生成导出文件。
- `PersonalDataService.processJobs` 定时处理任务。
- 每轮最多取 2 个 queued 任务。
- 任务开始处理时改为 running。
- Service 读取用户、资料和会话数据。
- Service 将数据写成 `account.json`。
- Service 将 JSON 打包为 ZIP。
- ZIP 上传到 MinIO。
- 成功后写入对象 key、completed 状态和 24 小时过期时间。
- 异常时写入 `EXPORT_FAILED`。
- 下载前按用户 ID 查询任务。
- 下载 UPDATE 只允许 completed、未消费、未过期的任务。
- UPDATE 行数为 1 时才生成 600 秒 MinIO 签名 URL。
- 重复点击、过期任务或他人任务不会生成下载链接。

### 3.5 账户注销与数据清理

- 注销请求先检查是否已有 queued/running 注销任务。
- 已存在任务时拒绝重复提交。
- 新请求插入 `account_deletion_jobs`。
- 请求受理后立即将用户设为 disabled。
- 请求受理后立即撤销全部认证会话。
- 定时任务删除头像和导出 ZIP 对象。
- 定时任务软删除用户资料、会话和消息。
- 定时任务软删除头像记录和导出任务。
- 成功后写入 `deleted_object_count`。
- 成功后将任务改为 completed。
- 任一步骤失败时写入 `DELETION_FAILED`。
- 失败时增加 `retry_count`。

### 3.6 管理员查询与前端真实模式

- `AdminUserController` 只查询用户摘要字段。
- 权限基类要求 operator/admin/superadmin 角色。
- 普通 user 直接返回 `403 FORBIDDEN`。
- 密码哈希、会话哈希和 CSRF 哈希不进入 DTO。
- 前端 Profile 页面轮询导出状态。
- 前端只允许一次性下载。
- 前端注销操作要求二次确认。
- real 模式下未接入模块显示未接入。
- real 模式下不伪造成功结果。

### 3.7 管理后台写操作

- `AdminManagementController` 统一要求 admin 或 superadmin。
- 每次成功写操作后写入 `operation_audits`。
- 审计记录保存操作者、trace ID、目标类型、目标 ID 和 action。
- 用户、工具和知识文档状态先执行白名单校验。
- 状态更新带 `is_deleted=FALSE` 条件。
- 软删除恢复通过资源类型白名单映射到固定表和主键。
- 资源类型不允许拼接任意表名。
- 知识文档上传限制 20 MB。
- `PersonalDataService.uploadKnowledge` 先写入 MinIO。
- MinIO 写入成功后创建 `knowledge_documents` 记录。

## 7. 管理后台写操作补充

管理后台已增加真实写接口，接口只允许 `admin` 和 `superadmin`，每次成功修改都会写入 `operation_audits`：

| 地址 | 方法 | 名称 | 功能 |
|---|---|---|---|
| `/api/admin/users/{id}/status` | PATCH | 用户状态更新 | active、locked、disabled 状态切换 |
| `/api/admin/users/{id}/sessions/revoke-all` | POST | 管理员撤销会话 | 撤销指定用户全部认证会话 |
| `/api/admin/tools/{name}/status` | PATCH | 工具状态更新 | 启用或停用已注册工具 |
| `/api/admin/knowledge/{id}/status` | PATCH | 知识文档上下线 | 更新文档状态 |
| `/api/admin/resources/{type}/{id}/restore` | POST | 软删除资源恢复 | 按白名单恢复受支持资源 |
| `/api/admin/knowledge` | POST multipart | 知识文档上传 | 文件写入 MinIO 私有桶并创建知识文档记录 |

real 模式下用户、工具、知识文档和软删除资源页面已调用上述接口，成功后重新加载页面；mock 模式保留演示行为。写接口统一携带 Cookie 和 CSRF Token，资源类型采用白名单校验。知识文档上传限制 20MB，并使用 MinIO 私有桶保存对象。
