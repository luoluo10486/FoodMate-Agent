# M1-1 账户、授权与个人数据

## 1. 实现状态

核心账户与个人数据服务已接入 PostgreSQL、QQ SMTP 和 MinIO。密码、会话、头像、导出、注销和 RBAC 后端接口可用；前端个人资料页已接入真实接口。管理后台目前只有用户查询具备真实后端接口，其余管理模块在 real 模式明确显示未接入，不再展示 mock 或伪造成功。

## 2. 接口清单

| 地址 | 方法 | 名称 | 功能 |
|---|---|---|---|
| `/api/auth/password-reset/request` | POST | 请求密码重置 | 生成一次性令牌并发送 QQ 邮件 |
| `/api/auth/password-reset/confirm` | POST | 确认密码重置 | 校验令牌并更新密码 |
| `/api/users/me/password` | POST | 修改密码 | 校验当前密码、更新密码并撤销全部会话 |
| `/api/users/me/sessions` | GET | 会话列表 | 查询当前账号的认证会话元数据 |
| `/api/users/me/sessions/{id}` | DELETE | 撤销单个会话 | 撤销指定认证会话 |
| `/api/users/me/sessions/revoke-all` | POST | 撤销全部会话 | 撤销当前账号全部认证会话 |
| `/api/users/me/avatar` | POST/DELETE | 头像上传/删除 | 使用 MinIO 私有桶保存或删除头像对象 |
| `/api/users/me/export` | POST | 创建导出任务 | 创建异步个人数据 ZIP 导出任务 |
| `/api/users/me/export/{id}` | GET | 导出状态 | 查询任务状态、过期时间和消费状态 |
| `/api/users/me/export/{id}/download` | POST | 消费下载链接 | 原子消费一次性下载资格并返回 10 分钟签名 URL |
| `/api/users/me/deletion` | POST | 申请注销 | 要求当前密码和 `DELETE_MY_ACCOUNT`，立即禁用账号并撤销会话 |
| `/api/admin/users` | GET | 管理员用户查询 | `operator`、`admin`、`superadmin` 查询脱敏用户摘要 |

## 3. 实现逻辑

- 密码使用现有 PBKDF2 格式；重置令牌只保存 SHA-256 哈希，15 分钟有效且只能消费一次。
- 会话撤销同时更新 PostgreSQL 的 `user_auth_sessions`，修改密码和注销会撤销全部会话。
- 头像限制 PNG/JPEG/WebP、2 MB；对象 key 写入 `user_avatar_assets`，桶保持私有。
- 导出任务由定时任务生成 ZIP 并写入 MinIO；下载接口使用条件更新确保链接只能消费一次，任务不存在、过期或重复消费返回明确错误。
- 注销先禁用账号，再由后台任务软删除用户关联数据并删除头像、导出对象，记录 `deleted_object_count`。
- 管理员用户查询只返回用户摘要，不返回密码哈希、会话哈希或 CSRF 哈希；普通 `user` 返回 `403 FORBIDDEN`。
- 前端 Profile 页面提供导出轮询、一次性下载和注销二次确认；真实模式下管理后台未接入的模块不会回退到 mock。

## 6. 当前边界

管理后台的 runs、tools、usage、knowledge、deleted 等页面尚无对应后端管理 API，因此尚不能宣称“管理后台全部页面真实接入”；当前 real 模式会显示 `API_NOT_CONNECTED`。导出 HTTP 端到端下载链路还需要在启动完整服务并准备可消费导出任务后补充浏览器级验证。
