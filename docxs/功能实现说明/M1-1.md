# M1-1 账户、授权与个人数据

## 1. 实现状态

核心账户与个人数据服务已接入 PostgreSQL、QQ SMTP 和 MinIO。密码、会话、头像、导出、注销和 RBAC 后端接口可用；前端个人资料页已接入真实接口。管理后台目前只有用户查询具备真实后端接口，其余管理模块在 real 模式明确显示未接入，不再展示 mock 或伪造成功。

## 2. 接口清单

| 地址 | 方法 | 名称 | 功能 |
|---|---|---|---|
| `/api/auth/password-reset/request` | POST | 请求密码重置 | 生成一次性令牌并发送 QQ 邮件 |
| `/api/auth/password-reset/confirm` | POST | 确认密码重置 | 校验令牌并更新密码 |
| `/api/users/me/password` | POST | 修改密码 | 校验当前密码、更新密码并撤销全部会话 |
| `/api/users/me/sessions` | GET | 会话列表 | 查询当前账号的认证会话元数据 |
| `/api/users/me/sessions/{id}` | DELETE | 撤销单个会话 | 撤销指定认证会话 |
| `/api/users/me/sessions/revoke-all` | POST | 撤销全部会话 | 撤销当前账号全部认证会话 |
| `/api/users/me/avatar` | POST/DELETE | 头像上传/删除 | 使用 MinIO 私有桶保存或删除头像对象 |
| `/api/users/me/export` | POST | 创建导出任务 | 创建异步个人数据 ZIP 导出任务 |
| `/api/users/me/export/{id}` | GET | 导出状态 | 查询任务状态、过期时间和消费状态 |
| `/api/users/me/export/{id}/download` | POST | 消费下载链接 | 原子消费一次性下载资格并返回 10 分钟签名 URL |
| `/api/users/me/deletion` | POST | 申请注销 | 要求当前密码和 `DELETE_MY_ACCOUNT`，立即禁用账号并撤销会话 |
| `/api/admin/users` | GET | 管理员用户查询 | `operator`、`admin`、`superadmin` 查询脱敏用户摘要 |
| `/api/admin/dashboard` | GET | 管理后台真实数据 | 查询运行、工具、模型用量、知识文档、软删除资源和操作审计数据 |

## 3. 实现逻辑

- 密码使用现有 PBKDF2 格式；重置令牌只保存 SHA-256 哈希，15 分钟有效且只能消费一次。
- 密码重置邮件使用中文 UTF-8 HTML 模板，同时提供纯文本降级内容；包含 FoodMate 标识、重置按钮、一次性令牌、有效期和防钓鱼提示。链接前缀由 `FOODMATE_WEB_BASE_URL` 配置。
- 会话撤销同时更新 PostgreSQL 的 `user_auth_sessions`，修改密码和注销会撤销全部会话。
- 头像限制 PNG/JPEG/WebP、2 MB；对象 key 写入 `user_avatar_assets`，桶保持私有。
- 导出任务由定时任务生成 ZIP 并写入 MinIO；下载接口使用条件更新确保链接只能消费一次，任务不存在、过期或重复消费返回明确错误。
- 注销先禁用账号，再由后台任务软删除用户关联数据并删除头像、导出对象，记录 `deleted_object_count`。
- 管理员用户查询只返回用户摘要，不返回密码哈希、会话哈希或 CSRF 哈希；普通 `user` 返回 `403 FORBIDDEN`。
- 前端 Profile 页面提供导出轮询、一次性下载和注销二次确认；真实模式下管理后台未接入的模块不会回退到 mock。

## 7. 管理后台写操作补充

管理后台已增加真实写接口，接口只允许 `admin` 和 `superadmin`，每次成功修改都会写入 `operation_audits`：

| 地址 | 方法 | 名称 | 功能 |
|---|---|---|---|
| `/api/admin/users/{id}/status` | PATCH | 用户状态更新 | active、locked、disabled 状态切换 |
| `/api/admin/users/{id}/sessions/revoke-all` | POST | 管理员撤销会话 | 撤销指定用户全部认证会话 |
| `/api/admin/tools/{name}/status` | PATCH | 工具状态更新 | 启用或停用已注册工具 |
| `/api/admin/knowledge/{id}/status` | PATCH | 知识文档上下线 | 更新文档状态 |
| `/api/admin/resources/{type}/{id}/restore` | POST | 软删除资源恢复 | 按白名单恢复受支持资源 |
| `/api/admin/knowledge` | POST multipart | 知识文档上传 | 文件写入 MinIO 私有桶并创建知识文档记录 |

real 模式下用户、工具、知识文档和软删除资源页面已调用上述接口，成功后重新加载页面；mock 模式保留演示行为。写接口统一携带 Cookie 和 CSRF Token，资源类型采用白名单校验。知识文档上传限制 20MB，并使用 MinIO 私有桶保存对象。

本轮验证：Docker 中 PostgreSQL、Redis、MinIO 均为 healthy；数据库 `FoodMate` 已有 32 张业务表；`AdminUserControllerRbacTest` 通过（2/2）；`AdminManagementControllerTest` 通过（2/2）；`M11AdminManagementE2ETest` 使用 `-Dfoodmate.local-e2e=true` 通过（1/1），覆盖真实注册、登录、Cookie/CSRF、管理员用户状态修改、工具启停、知识文档上下线、MinIO 私有桶上传与对象读取、会话撤销、软删除恢复、审计落库和普通用户越权拒绝；`M11ExportDownloadE2ETest` 通过（1/1），覆盖真实 MinIO ZIP 导出和一次性下载链接消费；`M11AccountDeletionE2ETest` 通过（1/1），覆盖账号注销、会话撤销、PostgreSQL 关联数据清理和 MinIO 对象清理。MinIO 私有桶在首次上传时自动创建，旧数据卷保留，新数据卷使用当前 `.env` 凭据。
