## Phase 1：前端静态 UI 原型

### F1-1. 创建前端项目骨架

状态：已完成。

- 建立 React + Vite + TypeScript + Arco Design 前端骨架，以 FoodMate design tokens、CSS Modules 和 mock 数据驱动页面。
- 建立页面、布局、组件、features、mock、类型和样式 token 边界；未接入真实后端或大型状态库。
- 验收事实：前端可启动并渲染首页，桌面与移动端基础布局可见。

### F1-2. 实现首页静态原型

状态：已完成。

- 实现顶部栏、会话侧栏、任务与推荐区域、Composer、Tools/Agents 状态及空态、加载态、错误态。
- 验收事实：首页与原型结构一致，mock 任务可展示，移动端不横向溢出。

### F1-3. 实现会话页静态原型

状态：已完成。

- 实现消息流、Agent 状态条、工具轨迹、引用、追问卡、确认卡和执行中停止入口。
- 验收事实：覆盖执行、追问、完成、失败、取消状态，轨迹与引用可展开且不依赖浏览器 alert。

### F1-4. 实现分析页静态原型

状态：已完成。

- 实现时间范围、营养汇总、趋势、目标对比、异常与缺失提示的静态页面及可替换图表数据结构。
- 验收事实：页面与原型一致，工具轨迹口径覆盖 `time_parser` 和 `database_query`。

### F1-5. 实现规划页静态原型

状态：已完成。

- 实现约束摘要、多日菜单、购物清单、预算、校验结果和确认保存界面。
- 验收事实：页面与原型一致，移动端可用，工具轨迹口径覆盖 `knowledge_search`、`database_query`、`plan_validator`。

### F1-6. 抽取共享 UI 组件

状态：已完成。

- 抽取会话、任务、Composer、状态条、工具轨迹、结果、引用、追问、确认、指标、计划、购物、空态和错误态组件。
- 验收事实：组件通过 props 复用，至少覆盖 normal、loading、disabled、error，业务组件保持 FoodMate Agent 语义。

## Phase 2：前端 Mock 交互

### F2-1. 添加假 Agent 事件回放

状态：已完成。

- 以本地 mock event sequence 模拟 `run.created` 至 `run.completed`、失败和取消事件，并在 hook/feature 层维护回放。
- 验收事实：会话页可播放完整 mock 任务；停止后不继续追加答案；Agent 状态、轨迹和答案流同步更新。

### F2-2. 添加 mock 追问流程

状态：已完成。

- 通过 `ClarificationCard` 模拟预算、忌口、目标等缺参追问，回答后从 `waiting_user` 回到 `planning`。
- 验收事实：追问内容明确、回答保留在消息流，且不接真实后端。

### F2-3. 添加 mock 确认流程

状态：已完成。

- 对饮食记录模拟确认、取消、修改及成功反馈，未确认前不展示已保存。
- 验收事实：不产生真实写入；失败和重试能力以当前前端实现与后续审计任务为准，不以历史 mock 记录替代。

### F2-4. 添加个人资料页 mock

状态：已完成。

- 新增 `/profile`，展示资料、营养目标、忌口、过敏原、单位偏好和修改密码入口；头像仅 mock 预览。
- 验收事实：mock 数据可渲染，保存只展示反馈，不调用真实接口。

### F2-5. 添加管理后台 mock 入口

状态：已完成。

- 新增 `/admin` 及 users、runs、tools、usage、knowledge、deleted 子路由和基于角色的 mock 视图。
- 验收事实：列表、筛选、详情、确认与权限反馈均有 UI 占位，不接真实管理接口。

## Java 后端已完成基线

### B3-1. 创建 Maven 模块化单体骨架

状态：已完成。

- 建立 Java 21、Spring Boot 3、Maven Wrapper、MyBatis-Plus、Flyway、PostgreSQL、Spring MVC/SSE 的模块化单体基础。
- 历史事实：建立 13 个模块骨架；当前目标已调整为 Java 控制面 + Python Agent Runtime。
- 验收事实：Maven 可构建，`foodmate-bootstrap` 可启动，模块依赖方向受约束。

### B3-2. 建立 shared 通用契约

状态：已完成。

- 建立 Snowflake ID、`ApiResponse<T>`、错误码、分页与 Trace 基础对象，外部 ID 采用字符串。
- 验收事实：错误响应包含 `code`、`message`、`details`、`request_id`、`trace_id`，并有基础单元测试。

### B3-3. 建立配置与环境文件

状态：已完成。

- 建立 local/dev/prod 配置结构，覆盖 PostgreSQL、Redis、Milvus、MinIO、RocketMQ、模型、RAG、SQL Agent、Tool 和 Audit 占位。
- 验收事实：不提交真实密钥，local profile 可启动或以 mock/stub 处理缺失外部依赖。

### B3-4. 建立统一异常与参数校验

状态：已完成。

- 建立全局异常处理、参数/业务/权限异常映射及工具、RAG、SQL、模型错误码前缀。
- 验收事实：参数、业务、内部错误有统一响应，不直接暴露堆栈。

### B3-5. 建立审计与软删除基础约束

状态：已完成。

- 建立软删除、恢复、默认过滤与审计字段约定。
- 验收事实：默认查询排除软删除数据，删除/恢复记录操作者与 trace，相关测试已建立。

### B4-1. 建立数据库迁移体系

状态：已完成。

- 建立 Flyway V1 迁移和回滚结构，PostgreSQL 为事务主库，主键使用应用层 Snowflake `BIGINT`，不以 pgvector 作为主向量库。
- 当前实现事实：V1 已建立 24 张核心表；`FlywayMigrationScriptTest` 校验核心表、软删除、索引、租户默认值、工具超时、注释和回滚脚本。
