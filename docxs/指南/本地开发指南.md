# FoodMate 本地开发指南

## 1. 当前边界

截至 2026-07-11，前端是 React/Vite 的静态 mock 原型；Java 是可编译、可启动的基础骨架；Python Agent Runtime 尚未创建。本指南中的“当前可执行”命令只覆盖前端和 Java。`local-stub` 不连接 PostgreSQL，也不代表认证、业务 API、AgentRun、Runtime Client、SSE、Tool 或 SQL Guard 已可用。

当前仓库没有 `agent-runtime/`、`pyproject.toml`、项目 Python 虚拟环境、Docker/Compose 或 CI 工作流；不要把下文目标态当作现有能力。

## 2. 当前可执行：前端

在仓库根目录执行：

```powershell
cd frontend
npm install
npm run dev
```

Vite 将在终端打印本地访问地址。当前服务只使用 `src/mock` 数据，不会连接 Java 后端。

可单独执行的检查命令：

```powershell
cd frontend
npm run build
npm run lint
npm run test
```

已知基线（2026-07-11）：`npm run build` 通过；`npm run lint` 因 35 个 warning 与 `--max-warnings 0` 失败；`npm run test` 因没有匹配 `src/**/*.{test,spec}.{ts,tsx}` 的测试文件失败。失败不能作为通过门禁处理。

## 3. 当前可执行：Java stub

根 `pom.xml` 要求 Java 21 和 Maven 3.9+；优先使用仓库 Maven Wrapper：

```powershell
.\mvnw.cmd -pl foodmate-bootstrap -am package
& java -jar '.\foodmate-bootstrap\target\foodmate-bootstrap-0.1.0-SNAPSHOT.jar' '--spring.profiles.active=local-stub'
```

`application.yml` 的默认 profile 已是 `local-stub`。该 profile 排除 DataSource 和 Flyway 自动配置，并将 `foodmate.runtime.stub-enabled` 设为 `true`。启动后在另一个 PowerShell 中检查：

```powershell
Invoke-WebRequest http://localhost:8080/actuator/health
```

上述 JAR 路径和参数已于 2026-07-11 实际验证：`package` 成功，`/actuator/health` 返回 HTTP 200 和 `UP`，进程随后已终止。日常前台运行时使用 Ctrl+C 正常停止。当前审计基线为 `.\mvnw.cmd clean verify` 通过，14 个 Maven Reactor 项目成功，Surefire 共 29 个测试、0 failure/error/skipped。该结果只证明现有骨架和测试，不证明目标态业务链路。

## 4. 真实 PostgreSQL 配置路径：当前未验证

`local`、`dev` 和 `prod` profile 均启用 DataSource 与 Flyway；当前仓库不提供数据库实例、容器编排或部署清单。本工作区没有对真实 PostgreSQL profile 的成功启动证据，因此本指南不提供未验证的启动命令。`application-local.yml` 包含可由 `DB_URL`、`DB_USERNAME`、`DB_PASSWORD` 覆盖的本地连接配置；不要把实际密码提交到 YAML、`.env` 或日志。

真实数据库启动、Flyway 执行和健康门禁必须在具备受控 PostgreSQL 后单独验证，且不代表业务 Repository、认证或 Runtime 集成已经实现。

## 5. 当前依赖与工作方式

前端依赖由 `frontend/package.json` 定义，开发入口是 Vite；其构建包含 TypeScript 检查。Java 根工程固定 Java 21、Spring Boot 3.5.16、MyBatis-Plus 3.5.16 和 Maven Enforcer。不要自行新增 Python、Docker 或 CI 命令来替代尚未存在的工程入口。

前端与 Java 目前可以分别运行，但没有真实 HTTP、SSE、token 或业务数据联调。开发 mock UI 时，必须把其结果视为原型行为；接入后端前，不能把它标记为端到端完成。

## 6. 目标环境/当前不可执行：Python Agent Runtime

目标态会新增独立 `agent-runtime/`、依赖锁定和项目虚拟环境。届时可在该目录安装依赖、启动 FastAPI Runtime 并运行 pytest；这些命令当前不可执行，因为对应目录、`pyproject.toml`、应用代码和测试均不存在。

目标 Runtime 只处理推理、编排、RAG、模型调用、质量校验和回答组装。它必须通过已批准的 V1 内部契约与 Java 通信，不能使用业务数据库凭据，也不能直接读写业务表、Tool 或 SQL 执行面。

目标 Python probe 路径固定为 `GET /foodmate/internal/health/live` 和 `GET /foodmate/internal/health/ready`。这两个端点当前不可执行，因为 Python Runtime 尚未创建；所有 Python health 调用必须保留完整的 `/foodmate/internal` 前缀。

## 7. 提交前最小检查

当前变更只涉及前端时运行 `npm run build`；涉及 Java 时运行 `.\mvnw.cmd clean verify`。若运行 lint 或前端测试，必须保留其当前失败事实，直到问题和测试文件被实际修复或补齐。未来 Python 命令和双运行时联调门禁只在目标工程落地后才可执行。
